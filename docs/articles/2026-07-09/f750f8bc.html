Vercel Agent: 本番環境の近くに配置できるエージェント

Vercel Agent を拡張しています。当初はアラート分類とプルリクエストレビューから始まりました。現在ではダッシュボードに統合され、本番環境を調査したり、プロジェクトについての質問に答えたり、承認後にアクションを実行したりできます。Vercel Agent はアプリをデプロイして実行するプラットフォームに組み込まれているため、本番環境で何か変更が発生すると、それが最初の対応者になります。ログ、メトリクス、デプロイを自動的に調査し、根本原因を特定し、ノートパソコンを開く前に修正案を提案します。Vercel Agent は独自のアイデンティティで動作し、デフォルトでは読み取り専用です。Vercel ダッシュボード、GitHub、CLI を通じてアクセスできます。

Vercel Agent があなたのためにすること

私たちは数か月間、本番デプロイメントで Vercel Agent を実行しています。典型的な調査はこのような流れです。

不良なデプロイが午後 11 時にリリースされ、チェックアウトエンドポイントが 500 エラーを返し始めます。オンコール エンジニアがログインする頃には、Vercel Agent は既にエラーを 4 分前にリリースされたデプロイまで追跡し、即座なロールバックを推奨しています。エンジニアがその計画を承認すると、Vercel Agent は前の本番デプロイメントにロールバックし、エンドポイントを修正するための PR の作成を開始します。アラートから問題解決まで：3 分以下です。

Vercel Agent でできること

Vercel Agent に直接タスクを指示することもできます。タスクを与えると、それが下準備をして、質問に答えるか、承認する修正を提示します。本番環境を独自に変更することはありません。

例えば、以下のことができます：

**プルリクエストをレビューする**。PR を確認するよう指示すると、パスした CI では検出されないパフォーマンス低下と危険な変更にフラグを立てます。

**コスト増加をトレース**。請求が跳ねた理由を尋ねると、キャッシュの代わりにすべてのリクエストでページをサーバーレンダリングするコード変更など、原因を特定します。承認があれば、修正を書いて PR を開きます。

**壊れたビルドを修正**。失敗したデプロイメントを Vercel Agent に指し示すと、ログを読み、失敗した設定を見つけ、更新の許可を求め、サンドボックスでビルドをテストします。

**リリース準備ができているか確認**。フィーチャーフラグについて尋ねると、コードとライブメトリクスを読み、ロールアウトが安全かどうかを教えます。

本番環境でのエージェント向けの新しいセキュリティモデル

アプリを修正できるエージェントは、それを壊すこともできます。本番環境の近くに配置するエージェントについては、最初の質問は次のようになります：デプロイ、設定変更、またはデータへのアクセスを許可することは安全ですか？

ほとんどの今日のエージェントについては、その答えはノーです。それは彼らがあなたの完全な権限を継承しているからです。あなたからの不適切なプロンプト、または混乱した サブエージェントは、あなたと同じ影響範囲を持ちます。選択肢は読み取り専用か継続的なアクセス、または慎重だが限定的か有能だが危険かのどちらかでした。

Vercel Agent は 3 つのものに基づいた新しい権限モデルを実装しています：エージェントが何であるか、何ができるのか、そしてそのコードがどこで実行できるかです。

Vercel Agent は独自のアイデンティティを持っています

ほとんどのエージェントはあなたとして機能します。1 つを接続すると、セッション全体で同じアイデンティティとアクセス権で機能します。エージェントができることとあなたができることの間に線はありません。

Vercel Agent は独自のプリンシパル `vercel-agent` として実行されます。これは 2 つの理由で重要です：

**属性**：エージェントのアクションは常にあなたのアクションと区別できます。すべての変更は、誰がそれを要求し、誰が承認し、Vercel Agent がそれを実行したかを記録します。実行したことは何も追跡不可能ではありません。

**権限**：別のアイデンティティは別の権限を意味しません。エージェントはあなたのアクセスを継承しません。計画承認によって明示的に付与されたもののみを取得し、それを指示している人が既に持っているもの以上のものは決してありません。

計画が権限です

エージェントを SDLC に統合することは通常、同じ方法で始まります。事前に広範なアクセスを付与することです。必要以上のものを、そして必要以上の期間付与します。
