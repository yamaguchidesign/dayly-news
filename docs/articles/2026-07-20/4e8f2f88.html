ランタイムログにキャッシュ理由が表示されるようになりました

ランタイムログに、リクエストが新鮮なキャッシュヒットにならなかった理由を説明するキャッシュ理由が表示されるようになりました。例えば、時間ベースまたはタグベースの再検証などです。キャッシュ理由を使用して、ミスをデバッグし、ヒット率を向上させることができます。

キャッシュ理由は、ISR、部分的なプリレンダリング、stale-while-revalidateなどのディレクティブを含むCache-Controlヘッダーを設定する関数など、CDNがキャッシュできるすべてのレスポンスに表示されます。すべてのリクエストで動的にレンダリングされたレスポンスにはキャッシュ理由がありません。

| キャッシュステータス | 考えられる理由 |

|---|---|

| MISS | Cold、Request collapsed、Error |

| BYPASS | Draft Mode、Prerender Bypass、Crawler |

| STALE | Time-based revalidation、Tag-based invalidation、Revalidation error |

| REVALIDATED | Tag-based deletion |

「ログ」タブを開いてリクエストを選択すると、キャッシュステータスと共に理由が表示されます。各ステータスと理由は、キャッシュステータスと理由のリファレンスエントリにリンクしています。

CLIから

キャッシュ理由は`vercel logs`、`vercel metrics`、およびVercelプラグインの`cdn-caching`スキルで利用可能です。

```

vercel logsでリクエストのキャッシュ理由を確認します：

vercel logs \

--request-id request-id \

--expand --json \

--project project \

--scope team

vercel metricsでトラフィック全体で各理由が発生する頻度を定量化します：

vercel metrics vercel.request.count \

--group-by cache_reason \

--since time \

--project project \

--scope team

Vercelプラグインのcdn-cachingスキルを使用して、キャッシングを大規模にデバッグします：

npx skills add https://github.com/vercel/vercel-plugin --skill cdn-caching

```

キャッシュステータスと理由について詳しく学びましょう。
