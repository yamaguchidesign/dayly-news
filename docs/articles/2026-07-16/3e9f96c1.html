Vercel Connect support in GitHub Tools

GitHub Toolsが新しい@github-tools/sdk/connectサブパスを通じて、Vercel Connectの第一級のサポートを提供するようになりました。長期間有効な個人アクセストークンを保存する代わりに、エージェントはコネクタから実行時に短期間有効でスコープ限定のGitHubトークンを生成します。保存、ローテーション、または流出の危険にさらされるシークレットはありません。

GitHubコネクタをプロジェクトにアタッチし、プリセットを選択します：

```

import { connectGithubTools } from @github-tools/sdk/connect ;

import { generateText } from ai ;

const tools = connectGithubTools ( github/my-connector , {

preset : code-review ,

} ) ;

const { text } = await generateText ( {

model : anthropic/claude-sonnet-5 ,

tools ,

prompt : Summarize open PRs on vercel-labs/github-tools. ,

} ) ;

```

カスタムエージェントでGitHub PRをレビューする

プリセットからのスコープ：

code-review、issue-triage、maintainerなどのプリセットはConnect スコープに自動的にマップされるため、トークンはツールセットに必要な権限のみを保持します。

eveで動作：

@github-tools/sdk/connect/eveからインポートし、agent/tools/内の1つのファイルがConnect バックアップの認証でフルツールセットを登録します。

マルチテナント対応：

installationId、repositories、またはscopesを呼び出しごとにオーバーライドして、特定のインストールをターゲットにするか、個別のリポジトリへのアクセスを限定します。

Vercelでのゼロ設定：

デプロイメントはOIDCトークンで自動的に認証されます。ローカル開発の場合は、vercel linkおよびvercel env pullを実行します。

カスタムツールファクトリの場合、connectGithubTokenは遅延トークンプロバイダーを返し、これを直接createGithubToolsに渡すことができます。

GitHubコネクタを作成し、ドキュメントを読むことで始めてください。デプロイの準備ができていますか？デプロイを開始するか、エキスパートに相談してください。
