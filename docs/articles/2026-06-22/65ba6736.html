Vercel Ship 2026 レビュー

10年間、Vercelはウェブがどのように構築されるかを形作ってきました。今、私たちはエージェントについても同じことをしています。次の10年で成功する企業は、最初からエージェント向けに設計されたインフラストラクチャ上に構築します。2,500人以上の人々がロンドンで開催されたVercel Ship 2026に集まり、まさにそれを実現しています。

エージェント向けインフラストラクチャ

Guillemoがアジェンティックインフラストラクチャに関するVercelのビジョンでShipをキックオフしました。それは、ソフトウェアを思考させることができるものを含め、何でもデプロイできる真のフルスタックプラットフォームです。

「私たちは思考できるソフトウェアをデプロイしています。」

- Guillermo Rauch Vercel CEO

エージェント向けインフラストラクチャは3つの部分で構成されています。

- Vercelは、コーディングエージェントがソフトウェアをデプロイする場所です。Claude CodeやCodexにどこにデプロイするかを聞くと、Vercelが返ってきます。Vercelはエージェントがどのように機能するかのために構築されているからです。

- Vercelは、独自のエージェントを構築・デプロイする場所です。本番環境でアプリケーションとエージェントを構築・実行するために必要なすべてのツールを提供します。セキュアに、規模に対応して。

- Vercel自体がエージェントによって自動化されています。Vercelは本番環境でアプリケーションを実行し、トラフィック、トレース、オブザーバビリティ、異常を処理します。そのデータにより、私たちのエージェントは自律的に調査するために必要なコンテキストを得られ、アラートだけでなくプルリクエストを表示できます。

Vercelはエージェント配信用です

次のキーノートでは、Tomが Agent Stackの各プリミティブについて説明しました。Hediは Vercel Connectを紹介し、エージェントが外部システムに接続するためのセキュアな方法を示しました。Sharは Vercelの新しいエージェントフレームワークである eveの完全なデモを行い、Agent Stackを数分で、単一のディレクトリで実装できることを示しました。

Agent Stack

ワークロードが何であれ、すべてのエージェントはモデルに接続し、多くのステップにわたってワークフローを実行し、それを有用にするシステムと人々に接続する必要があります。Vercelはこれら3つの機能をすべて構築するために必要なすべてのプリミティブを提供します。

- AI SDKは、任意のモデルを呼び出すための1つのAPIを提供し、ストリーミング、ツール呼び出し、構造化出力がすべてのプロバイダで同じように機能します。

- AI Gatewayは1つのエンドポイントから数百のモデルのリクエストをルーティングし、プロバイダがダウンした場合の自動フェイルオーバーを提供します。

- Workflow SDKは、自動再試行、状態の永続化、オブザーバビリティが組み込まれた、すべての実行を耐久性のあるものにします。

- Vercel Sandboxは、各エージェントに隔離されたmicroVMを提供し、本番環境に出荷する前にコードを実行・テストできます。

- Chat SDKにより、単一のコードベースからSlack、Discord、GitHubなど複数のプラットフォームにエージェントをデプロイできます。

Vercel Connect

Vercel Connectは、ロンドンで開催されたShip 2026でAgent Stackの最新のビルディングブロックとしてローンチされました。Connectは、環境変数に保存される長期間有効なプロバイダートークンを必要とせず、エージェントがそれらと連携するツール、データ、サービスへのセキュアなアクセスを提供します。アプリケーションやエージェントは、目の前のタスクにスコープされた一時的な認証情報をリクエストするため、リークの危険性がある永続的なシークレットが残されません。

エージェントをセキュアに接続します。Vercel Connectでプロバイダー認証情報を安全に保ち、アプリケーション全体でエージェントを認証・認可します。

eve

Vercelで数百のエージェントを構築した後、その下のアーキテクチャが同じ形をしていることに気付きました。eveは、その形をオープンソースフレームワークとして示すもので、Vercelでどのように本番環境のエージェントを構築、実行、スケールするかを表しています。

エージェントは単一のディレクトリに存在し、マークダウンで指示を、TypeScriptでツールを持ちます。耐久性のある実行、サンドボックス化されたコンピュート、承認、サブエージェント、評価がすでに配線されています。数分で本番環境のエージェントを配信できます。すべてのエージェントはディレクトリで、既に使用しているコマンドでスキャッフォルド、実行、デプロイできます。

Vercelはフルスタックアプリ配信用です

Guillemoはまた、追加のバックエンドフレームワーク、データベース、そしてマイクロサービスのサポートを含む、フルスタックアプリケーションをホストするVercelの能力を強調しました。
