AI Gateway がストリーミング文字起こしに対応

AI Gateway がストリーミング文字起こしに対応しました。これまで、文字起こしは完全なオーディオファイルが必要で、完全なトランスクリプトが単一の応答で返されていました。現在は、キャプチャされたままオーディオをストリーミングでき、モデルが生成するのと同時にトランスクリプトの更新を取得できます。これにより、ライブキャプションや音声入力などの用途で低レイテンシーを実現します。

ストリーミング文字起こしはベータ版で、AI SDK の streamTranscribe 関数を通じてストリーミング対応の文字起こしモデルで利用できます。以下の例は、生の PCM オーディオを openai/gpt-realtime-whisper にストリーミングし、到着したトランスクリプトのデルタを出力しています。結果のストリームは、部分的および最終的なトランスクリプトも含まれます。

```

import { experimental_streamTranscribe as streamTranscribe } from 'ai';

const result = streamTranscribe({

model: 'openai/gpt-realtime-whisper',

audio: audioStream, // ReadableStream Uint8Array | string

inputAudioFormat: { type: 'audio/pcm', rate: 24000 },

});

for await (const part of result.fullStream) {

if (part.type === 'transcript-delta') {

process.stdout.write(part.delta);

}

}

```

同じコードはプロバイダー全体で機能します。モデル文字列を xai/grok-stt またはその他のストリーミング対応文字起こしモデルに変更するだけです。ストリーミング文字起こしを使用すれば、エージェントに音声モードを簡単に追加できます。ユーザーの音声を文字起こしモデルにストリーミングし、ライブテキストをエージェントに渡します。エージェント自体は変わらず、引き続きテキストを受け取るため、任意のテキストベースのエージェントで機能します。エージェントが音声で応答する場合は、音声生成と組み合わせるか、リアルタイム音声を使用して双方向の完全な会話を実現できます。

AI Gateway でのストリーミング文字起こしの詳細については、ドキュメントをご覧ください。
