React Flightプロトコルの悪用と防御：RSCのデシリアライゼーション脆弱性

読了時間 27分 React、セキュリティ、コーディング Twitterで共有、LinkedInで共有 著者について Durgesh Rajubhai Pawarはハイパフォーマンスでアクセシブルなウェブアプリケーション開発に専念するフリーランス開発者です。彼は複雑な…Durgeshについてもっと詳しく ↬ メールニュースレター あなたの(素晴らしい)メール フロントエンド&UXに関する週間ヒント。182,000人以上に信頼されています。 スマートインターフェースデザインパターン、45レッスン+UXトレーニング Manuel Matuzovićによるアクセシブルなフロントエンドパターン 1,000万人の開発者を祝う Christine Vallaureとの Figma MCP AIインターフェースのデザインパターン、30レッスン+UXトレーニング Angular、React、Vue向けのカスタムウェブフォーム。あなたのバックエンド。 React Server Componentsはインタラクティブなuiをストリーミングするために独自のFlightプロトコルに依存していますが、この同じメカニズムは攻撃者が悪用できる強力なデシリアライゼーション脆弱性をもたらします。Durgesh Pawarは、CVSS 10.0の「React2Shell」脆弱性の背後にあるメカニクスを分解し、プロトコル操作がどのようにしてリモートコード実行につながる可能性があるかを示します。また、厳密なスキーマ検証からCSRF強化まで、React Applicationをこれらの構造的リスクから保護するための、実践的でランク付けされた防御セットについても説明します。

React Server ComponentsはブラウザにHTMLを送信しません。JSONも送信しません。サーバーコンポーネントがレンダリングされるときに実際にネットワークを通じて伝送されるのは、Flightと呼ばれるカスタムストリーミングプロトコルです。これは行区切り形式であり、独自の型システム、独自の参照解決、およびクライアント上で実行可能な動作を再構築するための独自のルールを持っています。ほとんどのReact開発者はネットワークタブを開いて、実際のFlightペイロードを見たことがありません。JSONフラグメント、ドル記号がプレフィックスされた参照、およびReactランタイムが静かにライブコンポーネントツリーに再組み立てするモジュールポインタが混在しているように見えます。フレームワークがそれを処理するため、誰もそれに疑問を持ちません。ほとんどのチームがその信頼が実際に何を意味するのかについて慎重に考えたかどうか確実ではありません。

2025年12月にCVE-2025-55182がリリースされた後、私はFlightプロトコルを分解し始めました。セキュリティコミュニティはそれをReact2Shellと呼び、理由があります。それはFlightデシリアライゼーション層にあるCVSS 10.0の認証されていないリモートコード実行脆弱性でした。Server Functionエンドポイントへの1つの作られたHTTPリクエストで、攻撃者はシェルアクセスを持っていました。認証情報は必要ありません。連邦サイバーセキュリティ・インフラストラクチャ庁（CISA）はそれを既知の悪用される脆弱性カタログに追加しました。Sysdigは、イーサリアムブロックチェーンを通じてファイルレスインプラントをデプロイする北朝鮮の国家支援行為者への野生での悪用を結びつけました。それはあなたの注目を得るCVEの種類です。

ソースコード（主にgetOutlinedModelとgetChunkで、実際に重要な解決ロジックが存在する場所）に時間を費やした後、React2Shellが単一の解析バグではないことに気づきました。それは症状でした。Flightはテキストのストリームから実行可能な参照、遅延ロードされたコンポーネント、サーバーRPCエンドポイント、および非同期状態を再構築します。これはデシリアライゼーションシステムです。攻撃面は単一の欠落したhasOwnPropertyチェックをはるかに超えています。

この記事は、Flightがネットワーク上でどのように機能するか、デシリアライゼーション脆弱性がどこにあるか、攻撃者がすでに何を悪用したか、そして何がまだ露出しているかについてカバーしています。これは、独自のServer Componentsのための、影響によってランク付けされた実践的な防御セットをもたらします：すべてのServer Actionでのスキーマ検証、server-onlyパッケージ、クロスサイトリクエストフォージェリ（CSRF）強化、およびTaint APIとWeb Application Firewalls（WAF）が提供するものの評価。
